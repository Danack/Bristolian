# Rooms Feature

Rooms contain files, links, annotations, and videos. Room tags can be created in the Room Management panel and then assigned to individual files, links, annotations, and videos. Tags are displayed alongside each entity and can be edited via an "Edit tags" button (only room tags created in Room Management can be assigned).

## Database

- `room` ‚Äì rooms
- `room_tag` ‚Äì tags belonging to a room (created in Room Management)
- `room_file` ‚Äì files in a room (UNIQUE on room_id, stored_file_id)
- `room_link` ‚Äì links in a room
- `room_annotation` ‚Äì annotations in a room
- `video` ‚Äì canonical YouTube video (youtube_video_id)
- `room_video` ‚Äì videos/clips in a room (full video or clip with start_seconds/end_seconds, parent_room_video_id for clips)
- `room_video_transcript` ‚Äì stored VTT transcripts per room_video (transcript_number, vtt_content)
- `room_file_tag` ‚Äì junction: which tags are on which room file
- `room_link_tag` ‚Äì junction: which tags are on which room link
- `room_annotation_tag` ‚Äì junction: which tags are on which room annotation
- `room_video_tag` ‚Äì junction: which tags are on which room video

Migrations: `db/migrations/35_room_entity_tags.php`, `db/migrations/36_room_videos.php`

## Cache-busting for room list API calls

When the client has just mutated content (e.g. uploaded a file, added a link/video, or saved tags), it must bypass HTTP caches so the refreshed list shows the new data. Room list GET endpoints support an optional `cacheBust` parameter for this:

- **Generated API helpers** in `app/public/tsx/generated/api_routes.tsx` (do not edit; generated by `php cli.php generate:typescript_api_routes`): `api.rooms.files`, `api.rooms.links`, `api.rooms.videos`, `api.rooms.annotations`, `api.rooms.file.annotations`, and `api.rooms.tags` accept an optional second argument `options?: { cacheBust?: boolean }`. When `cacheBust: true`, the request URL gets a `?_=Date.now()` query parameter so caches are bypassed.
- **Generator**: `src/Bristolian/CliController/GenerateFiles.php` ‚Äì `generateTypeScriptApiRoutes()` and `isRoomListEndpoint()` determine which routes get the optional param; `generateTypeScriptFromTree()` emits the conditional URL append.
- **Panels** pass `cacheBust: true` only when refreshing after a mutation (e.g. after `ROOM_FILES_CHANGED`, after save tags), not on initial load or on a generic Refresh button, so normal loads can still be cached.

## API

- GET `/api/rooms/{room_id}/files` ‚Üí `GetRoomsFilesResponse` (files: RoomFileWithTags[])
- GET `/api/rooms/{room_id}/links` ‚Üí `GetRoomsLinksResponse` (links: RoomLinkWithTags[])
- GET `/api/rooms/{room_id}/videos` ‚Üí `GetRoomsVideosResponse` (videos: RoomVideoWithTags[])
- POST `/api/rooms/{room_id}/videos` ‚Äì body `{ "url", "title?", "description?" }`
- POST `/api/rooms/{room_id}/videos/clips` ‚Äì body `{ "room_video_id", "start_seconds", "end_seconds", "description?" }`
- GET `/api/room-videos/{room_video_id}/transcripts` ‚Äì list transcripts (no room in path; transcripts are not secret)
- GET `/api/room-videos/{room_video_id}/transcripts/{transcript_id}` ‚Äì get one transcript VTT
- POST `/api/room-videos/{room_video_id}/transcripts/fetch` ‚Äì fetch transcript from YouTube (unofficial API)
- PUT `/api/rooms/{room_id}/videos/{room_video_id}/tags` ‚Äì body `{ "tag_ids": string[] }`
- GET `/api/rooms/{room_id}/annotations` ‚Üí `GetRoomsAnnotationsResponse` (annotations: RoomAnnotationWithTags[])
- GET `/api/rooms/{room_id}/file/{file_id}/annotations` ‚Üí `GetRoomsFileAnnotationsResponse` (annotations: RoomAnnotationWithTags[])
- PUT `/api/rooms/{room_id}/files/{file_id}/tags` ‚Äì body `{ "tag_ids": string[] }`
- PUT `/api/rooms/{room_id}/links/{room_link_id}/tags` ‚Äì body `{ "tag_ids": string[] }`
- PUT `/api/rooms/{room_id}/annotations/{room_annotation_id}/tags` ‚Äì body `{ "tag_ids": string[] }`

Only tag_ids that belong to the room (from Room Management) are accepted; invalid ids are filtered out.
